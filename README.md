# VTown Music 🎵

VTown Music is a dynamic Indian music recommendation system built entirely from scratch from first principles. By intentionally bypassing heavy-duty machine learning libraries like PyTorch or TensorFlow, this project implements recommendation logic using raw Python and vector mathematics to explore the fundamental mechanics of feature mapping and spatial similarity.

This repository documents the journey of building a recommendation engine from a blank CSV file up to a functioning 5-dimensional vector space model, focusing heavily on feature engineering and custom weight metrics.

## 📊 The Dataset (Hand-Curated)
Instead of relying on a generic, massive global dataset, this system utilizes a manually curated database of **125 songs** tailored specifically to the nuances of Indian cinema and music. Keeping the dataset compact allows for microscopic auditing of how individual audio metrics interact.

### Language Distribution:
*   **Tamil:** 40 songs
*   **Telugu:** 21 songs
*   **Hindi:** 21 songs
*   **Bengali:** 20 songs
*   **Kannada:** 6 songs

*Note: All core acoustic metrics (Energy, Danceability, Valence) were accurately sourced from [Musicstax](https://musicstax.com).*

## 🧠 Core Engineering Tasks

### 1. The 3D to 5D Feature Pivot
Initially, songs were represented as 3-dimensional vectors using purely acoustic metadata:
$$\vec{V} = [\text{Energy}, \text{Danceability}, \text{Valence}]$$

While mathematically sound, clustering purely on acoustic data yielded unexpected recommendations that lacked cultural or contextual logic. To map the "human factor" that actually hooks a listener, the vector space was expanded into a **5-dimensional model** by introducing normalized values for **Artist** and **Actor**:

$$\vec{V} = [\text{Energy}, \text{Danceability}, \text{Valence}, \text{Artist Frequency}, \text{Actor Frequency}]$$

*   **Frequency Weighting:** Artist and Actor metrics are dynamically computed based on their frequency of appearance within the database and scaled linearly from 0 to 1. 

### 2. The Math Behind the Similarity
Recommendations are generated by calculating the angular distance between song vectors using **Cosine Similarity**. 

$$\text{Cosine Similarity}(\vec{A}, \vec{B}) = \frac{\vec{A} \cdot \vec{B}}{\|\vec{A}\| \|\vec{B}\|}$$

#### The Tempo & Loudness Paradox Solved
One of the core findings of this project explains why two tracks with wildly different Tempos (BPM) or Loudness (dB) can return a near-perfect cosine similarity score. Because Cosine Similarity measures the **direction** of the vectors rather than their magnitude, songs with distinct tempos can share an identical structural ratio of Energy-to-Valence, causing them to align tightly in the multi-dimensional vector space.
